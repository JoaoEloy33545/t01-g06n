# The CSerVectorOPE
Incorporates the java logic in DvectorImpl java class implementing Dvector interface.
The src/main/resources/application.properties is of key importnce as the confifiguration base for regulating a diversity of operations aspects.
One example is the configuration of the web service port. In the example is adopted the 8080 port for validation. However the objective is to automate port generation by changing the configuration parameter quarkus.http.port from the initial 8080 to 0, ordering quarkus framework to generate the web service in a random port.

When adopting 0, be aware that it is nessary to somehow change client proxy to reference the right port. The approch suggest the adoption of a service registry for service location transparence and this way be ready to scale the distributed informatics system.

## Target artifacts
The directory target is generated by the Maven command

mvn install

An it can be eliminaded by

mvn clean

## Running the CSerDvector service

mvn quarkus:run

Validate the running web service by opening: http://localhost:8080/dvector?wsdl

^C to leave the CSerDvectorOPE running web service

## Note that the port needs to conform the instantiated service.
